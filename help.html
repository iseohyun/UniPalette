사용 설명서 및 알고리즘 정보

1. 통합 색상 추출 방법 안내

선형 색상 추출
HSL 색상 원통 공간 상에서 $360^\circ$의 각도 범위를 등간격 수식($i \times \frac{360}{count}$)으로 선형 분할하고, 지정된 명암 단계를 등간격으로 고정하여 정적 파레트를 만듭니다. 단순하지만 직관적인 기본 추출 형태입니다.
휘도 보정 (HSL 적응형)
HSL 공간 상에서 선형으로 색을 쪼개되, 각 각도별로 실제 광학적 상대 휘도(Relative Luminance)를 계산하여 명도(Lightness)를 적응형으로 조정합니다. 색상의 색조(Hue)가 바뀌더라도 인간 눈이 감지하는 시각적 밝기가 균일하게 유지되도록 최적화합니다.
색상 보정 (MacAdam Ellipse)
인간 시각의 **색 변별력 한계(MacAdam Ellipse)** 특성을 모델링하여 비선형 분할합니다. 눈이 둔감하게 구별하는 녹색($120^\circ$) 부근은 각도 간격을 크게 넓히고, 미세한 차이도 기민하게 감지하는 적색($0^\circ$) 및 청색($240^\circ$) 부근은 매우 조밀하게 쪼개어 진정한 인지적 등간격을 이끌어냅니다.
휘도 색상 보정 (HSL+MacAdam)
앞선 두 개의 고수준 보정을 동시에 결합합니다. MacAdam Ellipse에 근거해 Hue 각도를 시각적으로 최적화된 비선형 세그먼트로 분할한 후, 동시에 상대 휘도 보정을 가동해 명도를 세밀히 튜닝하여, 색상과 명도 모두에서 시각적으로 완전한 균일성을 획득하는 복합 알고리즘입니다.
인지적 등간격 (Oklab)
HSL 공간의 비선형 오차를 탈피하고 **Oklab 3D 벡터 인지 색 공간**상에서 다이렉트로 회전 연산을 처리합니다. 기준 색상을 sRGB → XYZ → LMS → Oklab으로 무손실 변환한 뒤, 무채색축($L_{ok}$)을 기점으로 $360^\circ / n$ 등분각만큼 기하학적 회전을 가하고, 이를 다시 감마 역보정과 클리핑을 거쳐 sRGB/HEX 코드로 직접 역산합니다. 붉은색 영역 등의 인지 변별 가중치 보정이 물리/수학적으로 완벽히 자동 반영되는 하이엔드 방식입니다.
CIEDE2000 고인지 색차 탐색
인간 시각의 색 정밀 인지 거리 연산 공식인 **CIEDE2000**을 기반으로 최적화 탐색을 수행합니다. 각 각도에 따라 기준 색상과 인지적으로 일치하는 색차 거리를 갖도록 명도를 계산하여 색조별 시각적 구별도를 극대화합니다.

2. 고정 색상 프리셋 정보

  • PowerPoint 기본색: 파워포인트 기본 디자인 도구에서 자주 쓰이는 대표 색상 7가지입니다. (7x1)
  • 무지개색 (MS Office): 마이크로소프트 오피스의 스탠다드 무지개 10선 색상표입니다. (10x1)
  • 구글 캘린더/행아웃색: 구글 서비스에서 사용되는 밝고 가시성 높은 24종의 캘린더 팔레트입니다. (8x3)
